解决过的问题

标签无法正确生成

  • 空白情况：在myblog/source 目录下没有tags文件夹，且文件夹里面没有index.md
  • 有页面情况但是无法显示所有标签：需要再markdown头部里面添加 —title: 标签 layout: tags —

这是 Hexo 的 **“布局 - 模板” 映射逻辑 **，核心是 Hexo 通过layout配置来匹配对应的渲染模板，具体原理如下：

  1. Hexo 的 “页面 - 布局 - 模板” 关联逻辑

Hexo 的页面渲染遵循「页面配置→布局名→模板文件」的映射规则：

  • 你在source/tags/index.md中写了layout: tags，表示这个页面要使用名为tags的布局；
  • Hexo 会自动去主题目录（themes/xxx/layout/）中找同名的模板文件（如tags.ejs/tags.html）；
  • 找到模板后，Hexo 会将全局数据（如site.tags标签列表）传递给模板，最终渲染出页面内容。
  1. 之前页面空白的原因

在你没加layout: tags时，Hexo 会用默认布局（通常是page布局）来渲染/tags页面：

  • 默认的page模板（如page.ejs）是通用的 “空白页面模板”，不会包含标签列表的渲染逻辑；
  • 所以页面只能显示标题，无法加载site.tags数据，呈现空白状态。
  1. 为什么加了layout: tags就生效？

添加layout: tags后，Hexo 会精准匹配到主题中的tags.ejs模板：

  • 这个模板里写了读取site.tags、渲染标签列表的逻辑；
  • Hexo 将标签数据传递给tags.ejs，模板就能把所有标签渲染出来，最终页面显示正常。

简单说：layout配置是 Hexo 的 “导航路标”，告诉 Hexo 该用哪个模板来渲染当前页面，不同模板对应不同的内容逻辑（比如tags模板负责渲染标签列表，post模板负责渲染文章）。

文章简介显示得太长

在 Hexo Starter 主题里，文章简介显示得太长，是因为它默认会直接截取文章开头的一大段文字。你可以通过下面几种方法来精准控制简介的显示内容。

方法 1：在文章里手动设置 description（推荐）

这是最精准的方式，你可以自己定义每篇文章的简介内容。

  1. 打开你想修改的 Markdown 文章

  2. 在文章开头的 Front-matter 区域（就是—包裹的部分），添加一行：

    yaml

  3. 保存后重新生成静态文件，主题就会优先显示你设置的这段文字，不会再自动截取大段内容。

方法2：修改主题配置文件（全局统一截取长度）

如果你想让所有文章的简介都自动保持统一的短长度，可以这样设置：

  1. 找到主题的配置文件 themes/hexo-starter/_config.yml

  2. 搜索 excerpt 相关的配置项，一般会有一个 length 参数

  3. 把长度值改小，比如：

    yaml

  4. 保存后重新生成静态文件即可。

方法3：在文章里用 <!-- more --> 分割符

在 Markdown 文章的任意位置插入 <!-- more -->，主题会只显示分割符之前的内容作为简介，后面的内容会被隐藏，读者需要点击 “阅读全文” 才能看到。

遇到应用上面方法不生效时

  1. 检查主题配置是否开启 description 优先

Hexo Starter 主题可能默认没有开启优先读取文章 description 的设置，需要手动在主题配置文件里开启：

  • 打开 themes/hexo-starter/_config.yml
  • 找到和文章列表相关的配置，比如 post_list 或 excerpt 模块
  • 确认是否有类似 use_description: true 的开关，把它设为 true

如果没有这个开关，说明主题默认不支持直接读取 description，需要用 <!-- more --> 分割符来控制简介。

  1. 修正 Front-matter 格式

你的配置里，description 用了双引号包裹，这在 YAML 里是允许的，但要注意符号是英文半角，且前后没有多余空格。
