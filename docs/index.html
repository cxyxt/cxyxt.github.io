没有撰写文档！

没有撰写文档！

3.0k 字 更新于 2026-01-06

你的文档页显示空白（字数为 0），核心原因是 **/docs页面对应的index.md文件中没有内容，且文档页的内容需要手动在 Markdown 文件中编写 **，具体逻辑和解决步骤如下：

一、核心逻辑：Hexo 的文档页 “不会自动生成内容”

Hexo 的文档页（/docs）本质是静态页面，它的内容来源是你在source/docs/目录下创建的 Markdown 文件（如index.md），不会自动引用其他文章 / 标签的内容—— 你当前的source/docs/index.md是空的（字数显示 “0 字”），所以页面没有内容。

二、解决步骤：手动编写文档的 Markdown 内容

1. 找到文档页对应的 Markdown 文件

进入 Hexo 根目录的source/docs/文件夹，打开index.md（这是/docs页面对应的源文件）。

2. 在index.md中编写内容

在index.md的头部配置后添加文档内容（Markdown 格式），示例：

三、验证效果

执行hexo clean && hexo g && hexo s重启服务，访问http://localhost:4000/docs/，此时页面会显示你在index.md中编写的 Markdown 内容，同时模板会自动渲染字数、更新时间、TOC 目录等信息。

总结

Hexo 的文档页不是 “自动聚合内容” 的页面，而是需要你手动在source/docs/目录下编写对应的 Markdown 文件来填充内容 —— 你当前的页面空白，就是因为source/docs/index.md中没有写任何内容。

文档不是只有一页，你可以创建多个文档（多级目录 + 多篇子文档），核心是通过source/docs/目录的文件结构 + 主题配置的导航逻辑来实现，具体步骤如下：

  2. 上下篇导航：模板的findDocNav函数会自动识别文档的层级，生成当前文档的 “上一篇 / 下一篇”（比如eva.md的上一篇是封边工艺首页，下一篇是pur.md）；
  3. 访问路径：每个文档对应独立的 URL（比如/docs/edge-sealing/eva/），互不干扰。

总结

通过 “文件结构分层 + 主题配置导航”，你可以创建任意数量的文档（支持多级子文档），每个文档都是独立的 Markdown 文件，模板会自动处理导航、TOC 目录、字数统计等功能 —— 本质是把source/docs/当作一个 “小型知识库目录”，手动组织文件即可实现多文档体系。
